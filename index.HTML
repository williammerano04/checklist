Website Build & Migration Checklist

Freshdesk Ticket Preparation

  • Check for assigned Ticket Number in Tracker
  • Login to Freshdesk
  • Assign the parent ticket to your name
  • Assign the child ticket to your name
  • Set Child ticket status to "In Progress"

CAP - Create Site

  • Login to CAP
  • Search client via GPID (e.g., TI SPARKL035) or Business Name
  • Action: Create Site using correct template from the ticket
  • Set subdomain using business name (e.g., SparklingHomes)
  • Go to builder and verify site creation
  • Check Builder

Builder Tasks

  • Start with fresh backup (new_build_JCR)
  • Edit Parent Ticket: Add Home Builder URL and Preview URL
  • Add Note to Child Ticket with:
    • Builder Link
    • Preview Link
  • Setup Content and Theme
  • Setup VCITA
  • Login to VCITA
  • Modify Template: Use info and content from the ticket
  • Business Address: Set to No (do not input address) do not include google maps
  • GBP Creation: Hide section from site
  • GBP Suspended: Hide section from site
  • GBP Verification: Leave section visible
  • *anything that is took from any site considered as page migration.example. Fb,IG, googel reviews, etc.
  • Check ticket for client logo, colors, preferences, etc.
  • Note: For long page names, create a folder for Services

Post Build Steps

  • Site backup end (end_build_JCR)
  • Internal QA(reminder: Do not Open Builder while under QA) Tracker
  • Parent Ticket:
    • Add Note
    • Insert QA Canned Response
    • Add Dev Note
  • Child Ticket:
    • Update Status from "In Progress" to "Resolved"
    • Add migrated # and total page count
    • Update Ticket

Reminder

  • Do not Open Website Builder if its in Extarnal QA Process
